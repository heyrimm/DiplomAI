# 1. 제품 또는 서비스의 목적 및 배경

## 1.1 문제 정의 — 공공외교의 "설계 공백"

### 1.1.1 제도와 현장의 간극

● 공공외교법(제2조)은 공공외교의 주체를 국가뿐 아니라 **지방자치단체 및 민간부문**으로 명시한다. 그러나 이들을 위한 기획 도구는 존재하지 않는다.
● 국회입법조사처는 「우리나라 공공외교의 추진체계 현황과 개선과제」(2022)에서 ▲공공외교의 **지역별·국가별 통계 시스템 미비** ▲**종합적 평가체계의 부재**를 지적한 바 있다.
● 대한민국시도지사협의회는 「지방자치단체 공공외교 추진실태 및 지원방안 연구」(2020)에서 중앙·지자체·민간의 공공외교 활동이 상호 조율 없이 수행되어 **유사·중복·편중·누락**이 발생함을 보고하였다.

### 1.1.2 데이터로 실증되는 공백

본 서비스가 KOICA ODA 지원실적과 KF 공공외교 사업 이력을 국가 단위로 교차 분석한 결과, **KOICA ODA 연 50억 원 이상 지원 국가 중 KF 데이터상 사업 이력이 없거나 최근 사업 이력이 2017년 이전에만 확인되는 국가 23개국**이 감지되었다.

| 상위 국가 | KOICA ODA (2023) | 전년비 | KF 공공외교 사업 |
|---|---|---|---|
| 에티오피아 | 339.9억 원 | +30.2% | 최근 확인 2016년 |
| 우간다 | 252.6억 원 | +14.8% | 최근 확인 2013년 |
| 몽골 | 224.3억 원 | +27.4% | 최근 확인 2015년 |

*※ 위 수치는 공공데이터포털 개방 데이터의 실계산 결과로, 서비스 화면에서 출처와 함께 직접 확인 가능하다. 이 공백은 어느 단일 기관 포털 화면에서도 보이지 않으며, 기관 간 데이터 융합으로만 드러난다.*

### 1.1.3 왜 지금까지 해결되지 못했나

● **데이터가 기관 사일로에 분산** — 외교부·KOICA·KF·한아프리카재단이 각기 다른 포털·데이터 구조·국가 표기·문자 인코딩(EUC-KR/CP949/UTF-8 혼재)으로 개방한다.
● **조사와 문서 작성의 단절** — 조사 결과가 실제 사업계획서 초안으로 이어지는 도구가 없어, 실무자는 여러 포털을 반복 검색한 뒤 별도로 문서를 작성해야 한다.
● **전문인력 부족** — 통계·데이터 분석 전담 인력이 없는 지자체·대학·민간 실무자는 분산 데이터를 스스로 융합하기 어렵다.

즉 문제는 데이터가 없는 것이 아니라, **분산된 공공데이터를 실무 의사결정과 사업 문서로 연결하는 도구가 없다는 것**이다.

## 1.2 대상 사용자 — 최우선 사용자를 한 명으로 특정

| 우선순위 | 사용자 | 처지 | DiplomAI 제공 가치 |
|---|---|---|---|
| **Primary** | **지자체 국제교류 담당 공무원** | 전담 인력·통계 전문성 부족, 예산 근거 요구가 강함, 외교·개발협력 기관의 내부 업무시스템은 소속 기관 외부자로서 이용 대상이 아님 | 근거 있는 대상국 선정 + KPI·리스크 포함 사업계획서 초안 |
| Secondary | 대학 국제처 실무자 | 한국학·학술교류 기획 | 한국학 인프라 기반 협력 대상 발굴 |
| Secondary | 민간 재단·NGO 실무자 | 제안서·공모 대응 | 제안서용 데이터 근거·타당성 진단 |

서비스의 1차 동선은 **Primary 사용자의 '대상국 선정 → 사업 문서화' 업무 흐름**을 기준으로 설계하고, 나머지는 동일 엔진을 공유하는 보조 사용자로 수용한다.

## 1.3 서비스의 목적

> **외교·개발협력 공공데이터만으로, 지자체·대학·민간 등 공공외교법상 주체 전체에게 '발견 → 진단 → 설계 → 실행 준비'의 AI 기반 사업 설계 도구를 개방하는 것.**

이는 공공외교 생태계의 통합적·체계적 구축 방향에 부합하는 공용 인프라 제안이며, 공공데이터 개방이 창출하는 정책적 가치를 동작하는 제품으로 실증한다.

# 2. 제품 또는 서비스의 기능 및 특징

## 2.1 핵심 사용자 흐름 — 하나의 업무 동선을 관통하는 코파일럿

DiplomAI는 기능 목록형 대시보드가 아니라, 좌측 내비게이션이 **업무 단계(① 국가 진단 → ② 사업 설계 → ③ 실행 검증)**로 구성된 코파일럿이다.

```text
[첫 화면: 공공외교 공백 국가 발견]
            ↓
[국가 종합 대시보드 — ODA·공공외교·안전·교류 기반 확인]
            ↓
 ┌──────────────────────┐
 │                      │
[AI 사업 추천]     [내 사업 타당성 진단]
 │   (탑다운)           │   (바텀업 · PDF 업로드 지원)
 └──────────┬───────────┘
            ↓  "이 사업으로 계획서" 원클릭
[근거 인용 사업계획서 초안 — KPI·리스크 포함]
            ↓
[현지 실행 준비 가이드 — KOTRA 근거]
```

> **[그림 1] 업무 단계형 내비게이션 및 전체 사용자 흐름 화면**

## 2.2 핵심 기능

### 2.2.1 공공외교 공백 자동 감지 (첫 화면)

● KOICA 국가별 ODA 지원실적 × KF 공공외교 사업 이력을 **전 국가 교차 스캔**하여, KOICA ODA 연 50억 원 이상 지원 국가 중 **KF 데이터상 사업 이력이 없거나 최근 사업 이력이 2017년 이전에만 확인되는 국가**를 탐지하고 ODA 규모순으로 노출한다. 이는 사업의 공식 중단 여부를 확정하는 지표가 아니라 추가 조사가 필요한 후보국을 우선순위화하는 신호다.
● 처리 흐름: 국가명 정규화 → 국가 단위 결합 → ODA 활동성 계산 → KF 사업 존재·최근성 확인 → 공백 후보 추출·정렬.
● 공백 감지는 정책 결론을 자동 확정하는 기능이 아니라, 사용자가 **추가 조사할 국가를 우선순위화하는 의사결정 지원 신호**로 설계하였다.
● 공백 카드 클릭 시 해당 국가 종합 분석으로 즉시 진입한다.

> **[그림 2] 공공외교 공백 국가 탐지 화면**

### 2.2.2 근거 인용 사업계획서 초안 생성

● 사업 배경 → 문제 정의 → 목표 → 대상·수혜자 → 주요 활동 → 추진 일정 → 예산 → **KPI** → **리스크(외교부 여행경보 자동 반영)** → 대응 방안 → 기대효과 구조의 초안을 생성한다.
● AI에 자유 작성을 맡기지 않는다. 국가별 통합 데이터 프로필을 먼저 구성해 프롬프트의 **근거 컨텍스트로 주입**하고, 제공된 데이터 범위 내 작성·근거 없는 수치 생성 금지·불확실 정보는 "현지 확인 필요" 표기를 강제한다.
● 본문의 인용 수치는 `[[339.9억 원|KOICA 국가별 지원실적 2023]]` 형식의 마커로 생성되며, 프론트엔드가 이를 파싱해 **하이라이트 + 마우스오버 출처 툴팁**으로 렌더링한다. "AI가 만든 문장"과 "공공데이터에서 가져온 근거"를 문장 단위로 구분·검증할 수 있다.
● Markdown 복사, 인쇄/PDF 출력을 지원한다.

> **[그림 3] 근거 인용 사업계획서 화면 (수치 하이라이트·출처 툴팁)**

### 2.2.3 양방향 진입 → 계획서 원클릭 파이프라인

● **(탑다운) AI 사업 추천**: 국가별 융합 데이터(ODA 추이·주요 분야·KF 이력·한국학 과정·재외동포·세종학당 수요·안전정보·지자체 교류 선례)를 근거로 사업 아이디어를 구조화 JSON으로 생성하며, 모든 추천 카드에 인용 데이터 출처를 명시한다.
● **(바텀업) 내 사업 타당성 진단**: 사용자가 사업 아이템을 입력하거나 **사업계획서 PDF를 업로드**하면, 분야 적합도·개발 수요·기존 ODA 채널·공공외교 연계·SDG 연계·국가별 위험 요인을 분석해 100점 스코어 + 강점·리스크·조정안을 제시한다. **점수는 공공데이터 신호의 규칙 기반 합산으로 계산하고, AI는 해석·설명만 담당**하여 설명 가능성을 확보하였다.
● 어느 쪽에서든 "이 사업으로 계획서" 버튼 한 번으로 계획서 생성으로 연결된다 — 발견·검증에서 실무 문서까지 끊기지 않는 동선.

```text
┌────────────────────────────────────────────┐

│  ① 데이터 로더(인코딩 자동 판별)           │
│  ② 국가명 정규화·국가별 통합 프로필        │
│  ③ 공백 감지·섹터 편차·전년비 실계산       │
│  ④ AI 근거 컨텍스트 구성·구조화 프롬프트   │
└─────────┬────────────────────┬─────────────┘
          ▼                    ▼
   공공데이터 (CSV/API)   Claude API (구조화 JSON)
   외교부·KOICA·KF·       추천·진단·계획서·
   한아프리카재단·KOTRA    실행 가이드 생성
```

● **역할 분리 원칙**: 공백 감지·전년비·섹터 편차·타당성 점수 등 계산 가능한 지표는 전부 코드로 계산하고, 생성형 AI는 해석·구조화·문서화에만 사용한다.
● **안정성**: 구조화 JSON 출력 강제, TTL 기반 응답 캐시(비용·지연 최적화), AI 실패·파싱 실패·키 미설정·데이터 누락에 대한 예외 처리.
● **보안**: IP당 레이트리밋, 보안 응답 헤더, 입력 화이트리스트 검증, 프롬프트 주입 완화 — 외부 공개 배포 대비.
● **배포**: Vercel(프론트) + Render(백엔드), 환경변수 키 관리, 프로덕션 빌드 검증 완료.

# 3. 외교부 및 산하기관 공공데이터 활용 방안

## 3.1 활용 데이터의 출처·내용·획득 방법

전체 데이터는 **공공데이터포털(data.go.kr)**의 Open API 인증키 발급 및 파일데이터 다운로드로 확보하였다. 별도 협약이나 비공개 내부 데이터는 사용하지 않아 **누구나 재현 가능**하다.

### 3.1.1 주최·주관 기관 데이터

| 기관 | 데이터 | 형태 | 서비스 내 활용 |
|---|---|---|---|
| **외교부** | 여행경보 | Open API | 국가 위험도 · **계획서 리스크 자동 반영** |
| **외교부** | 안전공지·경보 조정이력 | Open API | 국가별 실시간 안전정보 |
| **외교부** | 재외동포현황 | CSV | 공공외교 채널·교민 기반 지표 |
| **외교부** | 재외공관 현황 | API | 외교 인프라 지표 |
| **KOICA** | 국가별 지원실적 (1991–2023 시계열) | CSV | ODA 추이·전년비 · **공백 감지** |
| **KOICA** | 사업분야별 ODA 실적통계 | CSV | 연도별 전국 섹터 비중 (국가 데이터 부재 시 보정) |
| **KOICA** | 협력국 통합 개발지표 | CSV | 개발 수요 분석 · **섹터 편차 지역 평균 실계산** · 시뮬레이션 계수 |
| **KOICA** | SDG 성과지표 | CSV | 분야→SDG 연계 태깅 |
| **KF** | 융합 KF-공공외교·KOICA-ODA 사업정보 (**11,782건**) | CSV | 국가별 KF 사업 이력 · **공백 감지 핵심 소스** |
| **KF** | 해외대학 한국학 과정 운영현황 (**1,421건**) | CSV | 한국학 인프라·교류 수요 지표 |
| **한·아프리카재단** | 지자체-아프리카 교류협력 사례 (**387건**) | CSV | 아프리카 국가 한정 국내 지자체 선례 |

### 3.1.2 추가 연계 데이터

| 기관 | 데이터 | 활용 |
|---|---|---|
| KOTRA | 국가정보·해외시장뉴스 | 현지 실행 준비 가이드 근거(등록·인허가·통관·관례·통상 동향) |
| 세종학당재단 | 국가별 수강생 현황 | 한국어 학습 수요 지표 |

> **KF 데이터의 위상**: 본 서비스의 최대 차별점인 '공공외교 공백 감지'는 KF '융합 공공외교·ODA 사업정보'를 핵심 소스로 KOICA ODA 실적과 교차하여 산출된다. KF 데이터 없이는 성립하지 않는 기능으로, **KF의 데이터 개방이 창출하는 정책적 가치를 직접 실증**한다.

## 3.2 데이터 가공 — 단순 재검색이 아닌 파생지표 산출

### 3.2.1 기관 간 결합을 위한 전처리

● **인코딩 자동 판별**: 기관별 UTF-8·UTF-8-SIG·CP949·EUC-KR 혼재를 복수 인코딩 순차 판별 로더로 처리.
● **국가명 정규화**: `Myanmar / 미얀마 / 미얀마연방공화국 → Myanmar`처럼 기관마다 다른 국문·영문·약칭 표기를 매핑 테이블로 통일하여 국가 단위 결합을 가능하게 함.

### 3.2.2 파생지표

● **기관 교차 공백 신호**: KOICA ODA 활동성 × KF 사업 존재·최근성 → 공공외교 공백 후보.
● **지역 평균 대비 섹터 편차**: 국가 내부 분야별 지원 '비율'을 동일 지역 평균과 비교 — 총액 규모 차이에 왜곡되지 않는 비율 기반 방법론.
● **전년비 증가율**: `(당해연도 − 전년도) / 전년도 × 100`.
● **국가별 통합 프로필**: ODA·공공외교·한국학·재외동포·재외공관·여행경보·안전공지·교류 선례를 하나의 국가 객체로 통합, AI 근거 컨텍스트의 단위가 됨.

### 3.2.3 AI 인용 추적성 (traceability)

AI 생성물의 모든 인용 수치가 `[[수치|출처]]` 마커를 통해 화면에서 원천 공공데이터 출처와 연결된다. 심사·검토자는 생성 문서의 수치를 문장 단위로 원천 데이터와 대조할 수 있다.

## 3.3 확보의 지속성

● 여행경보·안전공지는 **실시간 Open API 연계**로 상시 최신성을 유지한다(키 미설정 시 안내 표기).
● CSV는 각 기관의 연 단위 정기 갱신 대상으로, **파일명 패턴 매칭 로드 구조**를 채택해 파일 교체만으로 갱신된다.
● 향후 기관 API 확대(KOICA 오픈데이터포털 API 등) 시 자동 갱신 파이프라인으로 확장 가능하다.

# 4. 기존 제품(서비스)와 차별성 및 독창성

## 4.1 근본 독창성 — 기관 교차 공백 감지

DiplomAI의 가장 강한 독창성은 **서로 다른 기관(KOICA × KF)의 데이터를 국가 단위로 교차하여, 단일 기관 포털 화면에는 보이지 않는 공공외교 공백 후보를 자동 탐지**한다는 점이다.

> "ODA 자원은 투입되고 있는데, 공공외교 채널은 상대적으로 비어 있는 국가는 어디인가?"

기존 포털은 각 기관의 데이터를 보여줄 뿐 이 질문에 답하지 못한다. 이는 데이터 '조회'가 아니라 **기관 간 데이터 '융합'으로만 생성되는 의사결정 신호**다.

## 4.2 기존 서비스와의 비교

| 기존 서비스 | 주요 기능 | 한계 | DiplomAI |
|---|---|---|---|
| KOICA 오픈데이터포털 | ODA 데이터 조회 | ODA 내부 분석에서 종료 | KF 등 타기관 데이터와 교차 → 공백 신호 생성 |
| KF 통계·사업정보 | KF 사업 조회 | 단일 기관 범위 | ODA와 결합, 조회→계획서→실행까지 연결 |
| 외교부 정보 서비스 | 외교·안전정보 제공 | 사업 기획 흐름과 분리 | 여행경보를 계획서 리스크에 자동 반영 |
| 범용 생성형 AI (ChatGPT 등) | 텍스트 생성 | 근거 데이터 불투명, 출처 추적 불가 | 공공데이터를 구조화해 근거로 주입, 수치별 출처 표시 |
| 일반 데이터 대시보드 | 시각화·조회 | 해석·문서화는 사용자 몫 | 발견→진단→계획서→실행의 업무 동선 완결 |

## 4.3 차별화 포인트 3가지

● **(a) 도메인** — ODA 데이터 분석에 그치지 않고 **공공외교 사업 설계**를 정면으로 다룬다. 국회입법조사처·시도지사협의회가 통계 시스템과 조율 체계의 미비를 공식 지적한(1.1.1) 바로 그 영역이다.
● **(b) 산출물** — 분석·추천 제시에 그치지 않고 **KPI·리스크 포함 실무 사업계획서 초안 + 현지 실행 준비 가이드**를 생성한다. '아이디어'가 아닌 '업무 문서'가 결과물이다.
● **(c) 완성도** — 아이디어 기획이 아닌 **동작하는 풀스택 제품 + 전부 공개 데이터 + 오픈소스** → 누구나 재현·검증 가능.

## 4.4 독창적 기술 요소

● **기관 교차 공백 감지 알고리즘** — 국가명 정규화 기반 KOICA×KF 전 국가 교차 스캔.
● **근거 강제·근거 가시화 생성 설계** — 제공 데이터 범위 내 작성을 강제하는 구조화 프롬프트 + `[[수치|출처]]` 마커의 하이라이트·툴팁 렌더링. AI 산출물을 문장 단위로 검증 가능.
● **규칙 계산과 생성형 AI의 역할 분리** — 점수·지표는 코드로 계산, AI는 해석·문서화 담당 → 설명 가능성 확보.
● **양방향 진입 원클릭 파이프라인** — 공백 발견(탑다운)과 내 아이템 진단(바텀업) 어느 쪽에서든 실무 문서까지 단절 없는 동선.
● **설계-실행 연결** — KOTRA 공공데이터 근거의 현지 실행 가이드로, 계획서 이후 '현지에서 어떻게 실행하나'까지 커버.

# 5. 기대효과

## 5.1 사회적 가치

● **지역 간 기획역량 격차 완화** — 전담 인력·분석 인프라가 부족한 지자체도 국가별 근거를 비교하고 사업 초안을 만들 수 있다. 시도지사협의회 연구(2020)가 보고한 지자체 공공외교의 조율·지원 한계에 대한 실질적 대응.
● **근거 기반(evidence-based) 행정** — 모든 AI 산출물이 원천 데이터 출처와 함께 표시되어 내부 검토·예산 심의에서의 검증 가능성을 높인다.
● **중복·편중 완화** — 기관 간 사업 이력을 함께 확인하여 기존 사업과의 중복을 검토하고, 비어 있는 국가·분야를 탐색 → 예산 중복 집행 예방.
● **SDG 연계** — 추천·계획서에 SDG 자동 태깅으로 SDG 17(글로벌 파트너십) 이행을 데이터로 지원.

## 5.2 정책적 활용

● 지자체 신규 국제교류국 발굴, 공공외교 사업 후보국 탐색, 대학 국제교류 프로그램 기획, 민간 재단·NGO 제안서 사전조사.
● **정책 공백의 실증 발견** — 공백 국가 목록 자체가 공공외교 자원 배분 재검토의 기초 자료.
● **데이터 개방 확대 역제안** — 개발 과정에서 확인한 개방 확대 후보(KF 사업별 예산·성과의 기계가독 개방, 지자체 국제교류 표준 데이터셋)를 기관에 환류.

## 5.3 운영 효율성 — 파일럿 검증 목표

```text
기존:  기관별 검색 → 엑셀 정리 → 비교 → 아이디어 회의 → 계획서 작성
DiplomAI:  국가 선택 → 공백·근거 확인 → 추천·진단 → 계획서 생성 → 실행 준비
```

정성 효과 나열에 그치지 않도록, 1단계 파일럿에서 다음 지표를 실측·검증한다.

| 측정 지표 | 검증 목표 |
|---|---|
| 대상국 조사 소요시간 | 파일럿 지자체 실측으로 단축률 검증 |
| 계획서 초안 작성 시간 | 기존 대비 단축률 검증 |
| 생성 수치의 출처 확인 가능 비율 | 인용 마커 커버리지 측정 |
| 파일럿 참여 지자체 수 | 10곳 확보 목표 |

## 5.4 일자리·경제적 효과

● 소수 인력 조직의 실질 기획 역량 증대 → 공공외교·국제개발 분야의 **데이터 기반 사업기획 직무** 저변 확대.
● 공공데이터 활용 창업 생태계에 재현 가능한 오픈소스 레퍼런스 제공.

# 6. 제품 및 서비스의 사업(창업) 계획

## 6.1 단계별 로드맵

### 6.1.1 1단계 — 공개·검증 (2026년 하반기)

● 웹서비스 무료 공개 및 오픈소스 저장소 정비로 초기 레퍼런스 확보.
● **Primary 사용자(지자체 국제교류 부서) 중심 파일럿 10곳 확보를 목표** — 광역·기초 지자체 국제교류 부서 대상 시연·인터뷰를 통해 5.3의 지표로 실무 적합성을 실측한다.
● 추천·계획서 생성 결과의 품질 고도화, 공공데이터 갱신 구조 강화, 기관 데이터 개방 확대 제안 환류.

### 6.1.2 2단계 — 기관용 수익화 (2027년)

● **B2G/B2I 기관 구독**: 기관 계정, 자체 데이터 업로드, 담당 교류국 저장, 변화 알림(여행경보 변경·신규 KF 사업 등록·공백 신호 발생), 심층 국가 리포트, 조직 내 계획서 버전 관리.
● **건별 컨설팅 리포트**: 신규 교류국 발굴·자매결연 후보 분석 등.
● 조달청 혁신제품·벤처나라 등록으로 공공조달 판로 확보.

### 6.1.3 3단계 — 확장 (2028년 이후)

● 지자체·대학·재단 대상 기관형 서비스 본격화, 공공기관 구축형(자체 데이터 결합 전용 환경) 공급.
● 데이터 확장: 국가별 한국 인식조사·문화행사·항공 연결성·무역투자 결합 → '설계 + 인식 모니터링' 통합 플랫폼화.
● 해외 공여기관 대상 화이트라벨 수출 모델 검토.

● 전부 공개 데이터 + 무료 티어 중심 기술 스택 → **낮은 운영비 구조**로 소규모 팀 지속 운영 가능.
● AI 응답 캐시·레이트리밋으로 운영 비용과 남용 위험을 통제.

# 맺음말 — 왜 DiplomAI인가

● **왜 필요한가** — 공공외교법상 주체의 절반(지자체·민간)에게 기획 도구가 없고, 기관 간 데이터 사일로가 유사·중복·편중·누락을 낳고 있다.
● **왜 이 데이터인가** — 외교부·KOICA·KF·한아프리카재단 개방 데이터의 국가 단위 융합만이 '공공외교 공백'이라는 새로운 정책 신호를 만들 수 있으며, 이는 데이터 개방의 가치를 직접 실증한다.
● **왜 실현 가능한가** — 이미 동작하는 풀스택 제품이 있다. 전부 공공데이터포털 개방 데이터로 구축되어 누구나 재현 가능하며, 오픈소스로 공개된다.
